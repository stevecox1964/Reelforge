## What was done this session

- **Diagnosed and fixed the "edits ignored at generation" bug.** Generation runs entirely off server-persisted project files (`run_stage` in `Python/scripts/studio/server.py:396` takes no body; reads `scene_plan.json` from disk). The only path flushing DOM edits to the server was `approvePlan()` (`web/studio/app.js`), which fires only on "Approve Plan". So edits made after approval, or before a Regenerate, silently used stale text.
  - Fix: scene-plan fields now **auto-save on blur**, mirroring the existing model-select auto-save. Added `saveScenePlan()` + a `focusout` delegated listener on `#scenes-list` in `initScenePlanCard`. No-ops while the JSON editor is open. Errors surface in the step-2 message line (not silent).
- **Diagnosed and fixed spinners that spun forever after completion.** `pollJob` cleared its timer and called `onDone()` but never stopped the infinite CSS spin animation; only the status badge changed.
  - Fix: gave the spinner a stable id (`${stage}-spinner`), added `markJobFinished(stage, status)` that strips `.spinner` and shows a static ✓ (green) / ✕ (red), and called it from `pollJob` on `completed` and `failed`/`interrupted`. Central fix — covers storyboards, clips, voiceover, review, and re-attached jobs.
  - CSS: added `.job-dot` / `.is-done` / `.is-error` in `web/studio/styles.css`, reusing existing green/red tokens.
- **README updated** — workflow step 2 now notes scene edits auto-save on blur.

## Decisions locked in this session

- **Auto-save on blur over a Save button** (user chose this). Why: a Save button has the same forget-to-click failure mode the user just hit; auto-save matches the existing model-dropdown pattern and removes the bug class entirely.
- **Spinner fix lives centrally in `pollJob`**, not per-stage handler. Why: one change covers every polling stage and future ones.
- **Progress bar stays visible after completion** (now with a ✓). Why: persistent "this really ran" record, consistent with the output `<details>` that already sticks around.

## Open questions

- **Step 1 (concept) post-create edits.** After `submitConcept()` the concept card stays in create/edit mode (it never switches to view) but there is no update endpoint, so editing idea/duration/aspect after creation does nothing; generation uses the manifest values set at create time. Decide: (a) switch the card to read-only view after create, or (b) add a backend update endpoint + save path so concept edits matter. Not yet started.
